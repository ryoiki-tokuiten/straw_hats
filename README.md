## Project

**Problem:ThreatSense AI-DVR** 

**Solution: system processes surveillance video using a dual-stream agentic architecture designed to convert raw footage into structured event memory while simultaneously evaluating potential risk.

The pipeline begins with a 2-minute video feed from the CCTV stream. This feed is sent to a vision-language model such as Qwen2.5-VL or Gemini. The model analyzes the clip and outputs timestamps of key moments with significant activity in JSON format. A Python script then uses FFmpeg to extract those key clips from the video. These clips represent the most informative parts of the two-minute segment and are used as the basis for further analysis.

The system then creates atomic reconstructions, which are concise narrative descriptions of what happened in each two-minute segment. An atomic reconstruction is intentionally minimal but descriptive enough that, when read together with previous reconstructions, it allows a human or model to fully visualize the evolving scene. These reconstructions are generated by a Narrative Builder Agent, which receives previous reconstructions along with the current key clips and produces a short description summarizing the event. The system already knows the timestamp range for each reconstruction, so the model does not need to output timestamps explicitly. Each reconstruction is stored in a database as video_id_timestamp → atomic_reconstruction.

At the same time, the system runs a Reasoning Agent in parallel. This agent receives the previous atomic reconstructions along with the latest two-minute video segment and analyzes the evolving narrative context to produce a risk score. The goal is to detect suspicious behavior patterns across time rather than just individual frames. Even if no suspicious activity is detected, the system still records atomic reconstructions for every segment, building a continuously updated narrative memory of the video stream.

To maintain real-time processing, the system handles potential race conditions when analysis takes longer than the two-minute input window. If a segment finishes processing late, the next segments continue to run in parallel using the best available context (either completed reconstructions or raw key clips). This delay is allowed to accumulate for up to three cycles (about six minutes). If the backlog grows beyond this threshold, the system switches to simplified “tool-less” versions of the agents that skip additional video queries and make decisions using only existing context, ensuring the pipeline continues operating without stalling.

All atomic reconstructions are also converted into embeddings, enabling semantic and multimodal retrieval later. The system stores only the past hour of reconstructions in the database, automatically trimming older entries to maintain efficiency. Through this design, the system compresses continuous video into a structured sequence of narrative events while simultaneously evaluating risk, enabling scalable and context-aware surveillance analysis.**
